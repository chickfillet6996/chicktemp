# ChickTemp Use Case Reports

The following use case report is based on the ChickTemp Farm Management Mobile Application. The system is used by the farm manager or poultry farm user to monitor environmental conditions, manage poultry batches, control devices, record farm activities, review analytics, receive alerts, and access support services.

| Use Case Name | Actors | Description | Preconditions | Flow of Events | Postconditions | Exceptions |
| --- | --- | --- | --- | --- | --- | --- |
| Create Account | Farm Manager/User | Allows a new user to register an account in the ChickTemp mobile application. | The user has not yet created an account. The device has an internet connection. | 1. User opens the sign-up screen.<br>2. User enters full name, email, password, and confirm password.<br>3. System validates the entered information.<br>4. System saves the account record to Firebase.<br>5. System redirects the user to the dashboard. | A new user account is created and stored in the system. | Duplicate Email: If the email already exists, the system displays an account error.<br>Invalid Input: If required fields are missing, the system asks the user to complete them.<br>Network Error: If Firebase cannot be reached, the system displays a connection message. |
| Login | Farm Manager/User | Allows an existing user to access the ChickTemp mobile application. | The user already has a registered account. The account credentials are valid. | 1. User opens the login screen.<br>2. User enters email address and password.<br>3. User may enable remember login.<br>4. System verifies the credentials.<br>5. System opens the dashboard after successful login. | The user is authenticated and gains access to the application. | Invalid Credentials: If the email or password is incorrect, the system displays a login error.<br>Network Error: If Firebase is unavailable, the system asks the user to check the connection. |
| Reset Password | Farm Manager/User | Allows the user to change the account password when forgotten. | The user has an existing registered email address. | 1. User opens the forgot password screen.<br>2. User enters the registered email address.<br>3. User enters and confirms the new password.<br>4. System validates the email and password.<br>5. System updates the saved password record. | The user's password is updated. | Email Not Found: If the email does not exist, the system displays an error.<br>Password Mismatch: If confirmation does not match, the system asks the user to re-enter the password. |
| Manage Profile | Farm Manager/User | Allows the user to view and update personal account details. | The user is logged in to the application. | 1. User opens the profile screen.<br>2. User selects edit profile.<br>3. User updates name, email, phone number, or profile photo.<br>4. System validates the information.<br>5. System saves the updated profile data. | User profile information is updated. | Invalid Details: If data is incomplete or invalid, the system displays a validation message.<br>Email Conflict: If the new email is already used, the system rejects the update. |
| Change Password | Farm Manager/User | Allows the logged-in user to update the account password. | The user is authenticated and has access to account settings. | 1. User opens change password.<br>2. User enters the required password details.<br>3. System validates the new password.<br>4. System updates the password record.<br>5. System confirms the password change. | The user account password is changed. | Invalid Password: If the password does not meet requirements, the system asks for a valid password.<br>Update Failed: If saving fails, the system displays an error message. |
| View Dashboard | Farm Manager/User | Provides an overview of active poultry batches, key farm metrics, and system status. | The user is logged in. At least one batch may exist in the system. | 1. User navigates to the dashboard.<br>2. System loads user batch records.<br>3. System displays available batches and summary information.<br>4. User selects a batch or reviews the dashboard overview. | The user views the current farm overview. | No Batch Found: If no batch exists, the dashboard shows an empty state.<br>Data Load Error: If batch data cannot be loaded, the system displays a message. |
| View Batch Dashboard | Farm Manager/User | Allows the user to view the details and current status of a selected poultry batch. | The user is logged in and has at least one poultry batch record. | 1. User selects a batch from the dashboard.<br>2. System opens the batch dashboard.<br>3. System displays batch details, grow-out day, total chickens, mortality count, and device options.<br>4. User chooses monitoring, controls, reports, or analytics. | The selected batch information is displayed. | Missing Batch: If the selected batch is no longer available, the system returns to the dashboard or displays an error. |
| View Real-Time Data | Farm Manager/User, ESP32 Sensor Device | Allows the user to view live environmental readings from the ESP32 sensor system. | The user is logged in. The ESP32 is powered and able to send sensor data. | 1. User opens the monitoring or environmental overview screen.<br>2. System retrieves the latest sensor data from Firebase.<br>3. System displays temperature, humidity, water level, and feeder level.<br>4. User reviews the current environmental condition. | The user sees the latest available sensor readings. | Sensor Offline: If the ESP32 is not sending data, the system displays offline or last known values.<br>No Reading: If a sensor cannot read data, the system displays `no_read` or an unavailable status. |
| View Daily Logs | Farm Manager/User | Allows the user to review recorded environmental log history. | The user is logged in. Environmental logs have been recorded. | 1. User opens the environmental logs or history section.<br>2. System retrieves saved temperature, humidity, water, and feeder readings.<br>3. System displays historical readings based on recorded time.<br>4. User reviews the previous environmental conditions. | Historical environmental records are displayed. | No Logs Found: If no logs exist, the system displays an empty log message.<br>Database Timeout: If logs cannot be retrieved, the system displays a loading error. |
| Control Devices | Farm Manager/User, ESP32 Sensor Device | Allows the user to control poultry house devices such as fan, water pump, feeder, and light. | The user is logged in and has selected a batch. ESP32 relay controls are connected. | 1. User opens the controls screen.<br>2. User selects a device control option.<br>3. User turns the device on or off.<br>4. System saves the control state in Firebase.<br>5. ESP32 reads the updated control state and applies it to the hardware. | The selected device control state is updated. | Hardware Offline: If ESP32 is offline, the command may be saved but not immediately applied.<br>Database Error: If Firebase cannot save the control state, the app displays an error. |
| Manage Feeding, Water, and Lighting Schedules | Farm Manager/User | Allows the user to create or update schedules for automated farm devices. | The user is logged in and has a selected batch. | 1. User opens feeder, water, or lighting settings.<br>2. User adds or edits a schedule.<br>3. User sets schedule time and active status.<br>4. System saves the schedule configuration.<br>5. System uses the schedule for device reminders or automation. | The schedule is stored and applied to the selected device configuration. | Invalid Time: If the schedule time is incomplete, the system asks the user to correct it.<br>Save Failed: If the schedule cannot be stored, the system displays an error. |
| Configure Automation Rules | Farm Manager/User | Allows the user to set automation behavior such as temperature thresholds and device rules. | The user is logged in and has access to the control or temperature settings screen. | 1. User opens automation or temperature settings.<br>2. User sets minimum and maximum temperature thresholds.<br>3. User configures related device behavior.<br>4. System saves the automation settings locally or in the device configuration.<br>5. System uses the settings for alerts and control guidance. | Automation and threshold settings are saved. | Invalid Threshold: If the minimum value is higher than the maximum value, the system asks the user to correct it.<br>Storage Error: If settings cannot be saved, the system displays a message. |
| Manage Poultry Batches | Farm Manager/User | Allows the user to create, update, or remove poultry batch records. | The user is logged in. | 1. User opens the batch management screen.<br>2. User adds a new batch or selects an existing batch.<br>3. User enters or edits batch name, start date, and total chickens.<br>4. System saves the batch record.<br>5. Dashboard updates the batch list. | Poultry batch records are created or updated. | Missing Details: If required batch information is incomplete, the system prevents saving.<br>Delete Error: If removing a batch fails, the system displays an error message. |
| Update Grow-Out Day | Farm Manager/User | Allows the system or user interface to show the current age or day label of a poultry batch. | A batch record exists with a start date or day label. | 1. User opens the batch dashboard.<br>2. System checks the batch start information.<br>3. System calculates or displays the current grow-out day.<br>4. User views the updated day label. | The batch day information is displayed or refreshed. | Invalid Date: If the start date is missing or invalid, the system displays the saved label or default value. |
| Record Mortality | Farm Manager/User | Allows the user to record chicken deaths for a selected batch. | The user is logged in and has selected a batch. The batch has a valid total chicken count. | 1. User opens mortality records.<br>2. User enters number of deaths, date, and notes.<br>3. System validates the mortality entry.<br>4. System saves the mortality record to Firebase.<br>5. System updates the batch mortality count and alive chicken count. | Mortality data is recorded and reflected in batch statistics. | Exceeds Remaining Birds: If deaths exceed the available chickens, the system rejects the entry.<br>Save Error: If Firebase cannot save the record, the system displays an error. |
| Add Event or Maintenance Report | Farm Manager/User | Allows the user to record farm events and maintenance activities. | The user is logged in and has selected a batch. | 1. User opens reports.<br>2. User chooses event or maintenance report.<br>3. User enters title, date, and description.<br>4. System validates the report details.<br>5. System saves the report under the selected batch. | The event or maintenance report is stored for future review. | Incomplete Report: If required fields are missing, the system asks the user to complete them.<br>Save Error: If the report cannot be stored, the system displays an error. |
| View Reports | Farm Manager/User | Allows the user to review saved event, maintenance, mortality, and environmental information. | The user is logged in and records are available. | 1. User opens the reports screen.<br>2. User selects the batch or report type.<br>3. System retrieves saved reports and records.<br>4. User reviews the report information. | Reports are displayed to the user. | No Records Found: If no reports exist, the system displays an empty state.<br>Load Error: If reports cannot be loaded, the system displays a message. |
| View Analytics | Farm Manager/User | Allows the user to review calculated farm performance and environmental analytics. | The user is logged in. Sensor logs, batch records, and mortality records may be available. | 1. User opens the analytics screen.<br>2. User selects a batch.<br>3. System loads environmental logs, mortality records, device configurations, and batch information.<br>4. System calculates averages, survival rate, mortality rate, and device-related summaries.<br>5. System displays analytics cards and charts. | The user views updated analytics for the selected batch. | Insufficient Data: If logs or records are missing, the system displays limited analytics.<br>Firebase Error: If analytics data cannot be loaded or saved, the system displays an error. |
| View Trends and Compare Batch Reports | Farm Manager/User | Allows the user to analyze environmental trends and compare batch performance. | The user is logged in and has historical records or multiple batch data. | 1. User opens analytics or reports.<br>2. User selects a batch or comparison view.<br>3. System retrieves historical logs and batch records.<br>4. System displays temperature trends, humidity trends, mortality rate, and survival rate.<br>5. User reviews the batch performance. | The user understands environmental and production trends. | Not Enough History: If historical data is too limited, the system displays available values only.<br>No Comparison Data: If only one batch exists, comparison may be unavailable. |
| Receive and View Alerts | Farm Manager/User | Allows the user to view system alerts, reminders, and warning notifications. | The user is logged in. Alert preferences and batch data are available. | 1. System checks sensor readings, thresholds, schedules, and device status.<br>2. System generates alerts for high temperature, low temperature, offline device, or schedule reminders.<br>3. User opens the alerts screen.<br>4. System displays active and historical alerts.<br>5. User may mark alerts as read or unread. | The user is informed of critical farm conditions and reminders. | No Alerts: If no alert is active, the system displays a no-alert message.<br>Load Error: If alerts cannot be loaded, the system displays an error. |
| Set Alert Preferences | Farm Manager/User | Allows the user to configure which alerts or reminders should be active. | The user is logged in and alert settings are accessible. | 1. User opens alert preferences.<br>2. User selects which alert types to enable or disable.<br>3. System saves the selected preferences locally.<br>4. System applies the preferences when generating alerts. | Alert preferences are saved and applied. | Storage Error: If local preferences cannot be saved, the system displays a message.<br>Invalid Setting: If a setting is unavailable, the system keeps the previous value. |
| Manage App Preferences | Farm Manager/User | Allows the user to adjust local app behavior such as remembered login and cached data preferences. | The user has access to the application settings or related preference screens. | 1. User opens the related preference screen.<br>2. User changes app preference settings.<br>3. System saves the preference locally using device storage.<br>4. System applies the updated setting on the next app use or refresh. | The app behavior follows the user's selected preferences. | Local Storage Error: If preferences cannot be saved, the system keeps the previous setting. |
| Access Support Services | Farm Manager/User | Allows the user to access help center content, policies, about app information, and customer support. | The user can access the support or profile section of the app. | 1. User opens support or information services.<br>2. User selects help center, app policies, about app, or contact support.<br>3. System displays the selected information or opens the support form.<br>4. User reviews information or submits a concern. | The user receives help information or submits a support concern. | Content Unavailable: If support information cannot be loaded, the system displays a message. |
| Contact Customer Support | Farm Manager/User, Support Email Service | Allows the user to send a support concern to the system support contact. | The user is logged in or has contact details available. The device has an internet connection. | 1. User opens contact support.<br>2. User enters subject and message.<br>3. System attaches user contact information.<br>4. System stores the support ticket in Firebase.<br>5. System sends an email copy through the support email service. | The support ticket is recorded and sent to the support contact. | Missing Message: If subject or message is empty, the system asks the user to complete the form.<br>Email Service Error: If the email service fails, the system may still save the ticket or display an error.<br>Network Error: If no connection is available, the ticket cannot be submitted. |
| Logout | Farm Manager/User | Allows the user to end the current app session. | The user is currently logged in. | 1. User selects logout.<br>2. System clears the active session.<br>3. System returns the user to the login screen. | The user is logged out of the application. | Session Error: If logout fails, the system keeps the user on the current screen or displays a message. |
