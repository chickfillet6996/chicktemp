# ChickTemp Use Case Reports

The following table presents the important use cases of the ChickTemp Farm Management Mobile Application. These use cases focus on the main functions needed by the farm manager: account access, dashboard monitoring, batch management, device control, environmental thresholds, mortality recording, analytics, alerts, and support.

| Use Case Name | Actors | Description | Preconditions | Flow of Events | Postconditions | Exceptions |
| --- | --- | --- | --- | --- | --- | --- |
| Create / Login Account | Farm Manager/User | Allows the user to create an account or log in to access the ChickTemp mobile application. | The user has a valid email address. The device has an internet connection. | 1. User opens the app.<br>2. User creates an account or logs in.<br>3. System validates the account and opens the dashboard. | The user gains access to the application. | Invalid login or network error: System displays an error message. |
| Manage Profile | Farm Manager/User | Allows the user to update account details such as name, email, phone number, profile photo, and password. | The user is logged in to the application. | 1. User opens profile.<br>2. User edits account details.<br>3. System validates and saves changes. | The user's account information is updated. | Invalid details or save error: System asks the user to correct or retry. |
| View Dashboard and Monitoring | Farm Manager/User, ESP32 Sensor Device | Allows the user to view the farm dashboard, selected batch status, and real-time environmental readings. | The user is logged in. The ESP32 device is connected or has previously sent sensor data. | 1. User opens dashboard.<br>2. User selects a batch.<br>3. System displays sensor readings and batch summary. | The user views the current farm and environmental status. | Sensor offline or no batch: System shows last reading, offline status, or empty state. |
| Manage Poultry Batches | Farm Manager/User | Allows the user to create, edit, delete, and review poultry batch records. | The user is logged in. | 1. User opens batch management.<br>2. User adds or edits batch details.<br>3. System saves the batch record. | Poultry batch records are created, updated, or removed. | Missing details or delete error: System prevents the action and shows a message. |
| Control Devices and Automation | Farm Manager/User, ESP32 Sensor Device | Allows the user to control and configure devices such as ventilation fan, water pump, feeder, light, and schedules. | The user is logged in and has selected a batch. ESP32 relay controls are available. | 1. User opens controls.<br>2. User updates device or schedule settings.<br>3. System saves the command for the ESP32. | Device controls and automation settings are updated. | Hardware offline or invalid schedule: System delays the command or asks for correction. |
| Record Mortality | Farm Manager/User | Allows the user to record chicken deaths for a selected poultry batch. | The user is logged in and has selected an existing batch. | 1. User opens mortality records.<br>2. User enters deaths, date, and notes.<br>3. System saves the record and updates the count. | Mortality records are saved and reflected in the batch statistics. | Invalid count or save error: System rejects the entry or shows an error. |
| View Reports and Analytics | Farm Manager/User | Allows the user to review farm reports, environmental history, trends, mortality rate, and survival rate. | The user is logged in. Batch records, sensor logs, or mortality records are available. | 1. User opens reports.<br>2. User selects a batch.<br>3. System displays charts, averages, and rates. | The user reviews farm performance and environmental trends. | Insufficient data or load error: System shows limited results or an error message. |
| Manage Alerts and Preferences | Farm Manager/User | Allows the user to view critical alerts and configure alert or app preferences. | The user is logged in. Sensor data, schedules, or thresholds are available. | 1. System checks readings and thresholds.<br>2. User opens alerts.<br>3. User reviews alerts and updates preferences. | The user is informed of important farm conditions and preferences are saved. | No alerts or save error: System shows status or keeps previous settings. |
| Access Support Services | Farm Manager/User, Support Email Service | Allows the user to view help information, app policies, and contact customer support. | The user can access the support or profile section of the app. | 1. User opens support.<br>2. User views help or submits a concern.<br>3. System saves the ticket and sends a copy. | The user receives help information or submits a support request. | Incomplete form or email error: System asks for details or saves the ticket only. |
| Manage Environmental Thresholds | Farm Manager/User | Allows the user to set minimum and maximum temperature limits used for monitoring, alerts, and environmental control guidance. | The user is logged in and has selected a poultry batch. | 1. User opens threshold settings.<br>2. User enters temperature limits.<br>3. System validates and saves the settings. | Environmental threshold settings are saved and used by the system. | Invalid threshold or save error: System asks for correction or keeps previous values. |
| Logout | Farm Manager/User | Allows the user to end the current session and return to the login screen. | The user is currently logged in. | 1. User selects logout.<br>2. System ends the session and returns to login. | The user is logged out of the application. | Session error: System shows a message or keeps the session active. |
